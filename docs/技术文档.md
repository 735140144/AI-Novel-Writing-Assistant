2. 页面级 PRD 骨架（可直接给 UI/前端开工）

2.0 全局规范（所有页面通用）

布局：左侧域导航 + 顶部运行条 + 中央主画布 + 右侧详情抽屉（按需）。
状态：统一支持 loading / empty / error / stale / success 五态。
任务状态：统一支持 created / queued / running / paused / failed / canceled / completed / recovered。
主按钮层级：每屏仅 1 个主 CTA，其他动作下沉二级。
审计：所有“继续/恢复/自动通过/配置保存”动作写入操作日志。
2.1 启动看板 /launchpad
页面目标
让用户 10 秒内知道“现在最该做什么”。

核心模块

生产脉冲卡：进行中、待处理、可继续、失败告警。
快速启动区：自动开案、手动开案、导入素材。
当前优先项目卡：单卡显示最优先恢复对象。
最近项目流：压缩展示多个项目状态。
恢复中心抽屉：中断任务批处理恢复。
核心数据字段
projectId, title, stage, progress, healthState, checkpoint, updatedAt, recommendedAction, riskLevel

核心动作
立即继续 / 查看上下文 / 打开执行枢纽 / 批量恢复 / 稍后处理

接口建议

GET /api/launchpad/summary
GET /api/launchpad/projects?limit=20
GET /api/recovery/pending
POST /api/recovery/resume
验收标准

首屏不超过 3 块核心信息组。
用户可在 2 次点击内进入任一项目恢复点。
恢复操作支持单个与批量。
异常任务必须有明确下一步建议。
2.2 作品仓库 /works
页面目标
统一管理作品资产和阶段进度，支持检索与批量操作。

核心模块

过滤区：状态、创作类型、更新时间、标签。
视图切换：卡片视图/表格视图。
作品卡片：简介、阶段、进度、健康度、建议动作。
批量工具条：导出、归档、标签管理。
核心数据字段
workId, title, synopsis, type, status, stage, progress, chapterCount, characterCount, tokenUsage, lastTaskState

核心动作
进入项目 / 预览 / 导出 / 归档 / 批量标签

接口建议

GET /api/works
POST /api/works/batch-export
POST /api/works/batch-archive
PATCH /api/works/{id}/tags
验收标准

首次加载后 1 秒内可操作筛选。
卡片默认展示不超过 6 个关键字段。
删除/高风险动作必须二次确认。
支持分页与无限滚动二选一配置。
2.3 创作实验室 /studio
页面目标
在单页内完成“问题诊断 -> 方案生成 -> 资产回填 -> 下一步推进”。

核心模块

左栏线程：会话列表、新建、归档。
中栏会话：对话、摘要、候选方案。
右栏推进面板：准备度清单、缺口补齐、绑定资产。
回填中心：把会话结果写回作品字段。
核心数据字段
threadId, focusTopic, readinessScore, missingItems, decisions, assetBindings, nextPrompt

核心动作
继续此方向 / 重新生成 / 生成备选 / 应用到项目 / 生成确认清单

接口建议

GET /api/studio/threads
POST /api/studio/threads
POST /api/studio/messages
POST /api/studio/apply
GET /api/studio/readiness
验收标准

三栏布局可在 1440 宽度一次看全核心信息。
会话结果可一键回填到作品资产。
准备度缺口必须可定位到具体字段。
每次推进都写入“决策摘要”。
2.4 素材拆解台 /analysis-lab
页面目标
将外部文档转为结构化创作资产。

核心模块

创建向导：选择文档、选择版本、模型参数。
任务列表：草稿/排队/运行/成功/失败/归档。
结果工作区：章节梳理、人物关系、时间线、主题线。
回流动作：发送到创作实验室/作品资产。
核心数据字段
analysisId, sourceDocId, docVersion, modelPolicy, temperature, tokenLimit, timelineEnabled, status, resultSummary

核心动作
新建拆解 / 重跑 / 对比版本 / 回流到项目

接口建议

POST /api/analysis
GET /api/analysis
GET /api/analysis/{id}
POST /api/analysis/{id}/rerun
POST /api/analysis/{id}/push-to-work
验收标准

新任务创建流程不超过 4 步。
结果区支持结构化分栏阅读。
失败任务可复制参数重试。
回流动作可追踪来源版本。
2.5 执行枢纽 /ops/jobs
页面目标
统一监控所有执行任务，快速定位异常并恢复推进。

核心模块

运行指标条：运行中、排队中、失败、24h 完成。
多维筛选器：类型、状态、排序、关键字、仅异常。
任务列表区：核心状态+建议动作。
详情侧栏：阶段轨迹、模型调用、心跳、重试信息。
核心数据字段
jobId, jobType, ownerWorkId, status, stage, currentStep, heartbeatAt, retryCount, modelUsed, inputTokens, outputTokens, checkpoint

核心动作
打开来源 / 查看检查点 / 进入实验室诊断 / 恢复执行 / 取消任务

接口建议

GET /api/jobs
GET /api/jobs/{id}
POST /api/jobs/{id}/resume
POST /api/jobs/{id}/cancel
GET /api/jobs/metrics
验收标准

“仅异常”一键过滤。
任务详情必须显示下一步建议。
恢复动作有操作回执。
大列表下滚动性能稳定。
2.6 流程督导台 /ops/supervision
页面目标
面向运营治理，管理优先级、告警、自动推进与人工介入。

核心模块

分区泳道：待校验、异常、待处理、自动推进、已替代。
优先级队列：P0/P1/P2 明确分层。
跟进详情：阻塞原因、建议动作、通道触达记录。
批量治理：批量恢复、批量标注、批量转派。
核心数据字段
followupId, priority, partition, reasonCode, status, modelRoute, channelDelivery, slaDeadline, operator

核心动作
恢复 / 标注已处理 / 转派 / 查看来源 / 批量操作

接口建议

GET /api/supervision/board
GET /api/supervision/{id}
POST /api/supervision/{id}/resolve
POST /api/supervision/batch
GET /api/supervision/delivery-log
验收标准

P0 项默认置顶显示。
自动推进日志与人工待处理分开视图。
每条异常必须有标准化原因码。
批量动作可撤销或回滚。
2.7 模型编排台 /ai/policies
页面目标
按任务角色编排模型策略，保证质量、稳定性与成本平衡。

核心模块

策略模板区：快速套用策略集。
角色矩阵区：规划/写作/审校/修复/摘要/考据等角色。
结构化回退区：主模型失败后的兜底策略。
检测与版本区：生效检测、版本保存、回滚。
核心数据字段
policyId, roleKey, provider, model, protocol, outputFormat, temperature, maxTokens, fallbackPolicy, validationState, version

核心动作
保存策略 / 批量套用 / 对比版本 / 回滚 / 运行自检

接口建议

GET /api/ai/policies
PUT /api/ai/policies/{id}
POST /api/ai/policies/batch-apply
POST /api/ai/policies/validate
POST /api/ai/policies/{id}/rollback
验收标准

角色与模型关系一屏可比较。
协议与输出格式不兼容时实时告警。
支持策略版本回滚。
检测结果可追溯到策略版本号。
2.8 平台控制台 /platform/config
页面目标
集中管理平台运行参数、自动化偏好、通知通道与模型供应商。

核心模块（Tab 化）

运行参数：超时、并发、默认阈值。
自动化策略：审批门配置、风险级别授权。
通知通道：钉钉/企微 webhook、token、用户映射。
供应商管理：API、模型清单、测试连接、余额刷新。
核心数据字段
configKey, configValue, riskLevel, channelType, webhookStatus, providerStatus, rateLimit, balanceState

核心动作
保存设置 / 测试连接 / 刷新模型 / 刷新余额 / 新增供应商

接口建议

GET /api/platform/config
PUT /api/platform/config
POST /api/platform/channels/test
GET /api/platform/providers
POST /api/platform/providers/{id}/refresh-models
验收标准

每个 Tab 独立保存，不互相污染。
敏感字段默认脱敏。
保存后明确返回影响范围。
通道测试结果可审计。
你可以直接让团队开工的最小落地顺序

第一阶段：启动看板 + 作品仓库 + 执行枢纽。
第二阶段：创作实验室 + 素材拆解台。
